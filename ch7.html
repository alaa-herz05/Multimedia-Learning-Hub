Pixels & Resolution
🖼️
Digital Image Fundamentals
الشرح بالعربي الصورة الرقمية تتكون من عدد كبير من عناصر الصورة (Pixels). كلما زاد عدد البكسلات زادت الجودة والدقة. طريقة تخزين الصور تُسمى Bitmap — وهي نفس الطريقة التي يُخزَّن بها محتوى الشاشة في ذاكرة الفيديو.
  • A digital image consists of many picture elements = pixels
  • Number of pixels determines image quality / resolution
  • Higher resolution → always better quality
  • Bit-map stores image data the same way monitor contents are stored in video memory
📐
Resolution Formula & Pixel Depth
الشرح بالعربي دقة الصورة = العرض × الارتفاع. عمق الألوان (Pixel Depth) = عدد البتات لكل بكسل → عدد الألوان = 2^n. مثال امتحاني: صورة 100×100 بكسل، عمق 8 بت → الحجم = 100×100×8 = 80,000 بت = 10KB.
Resolution = W × H
  • Width × Height in pixels
  • e.g. 640×480, 1920×1080
Pixel Depth
  • 8, 16, 24, or n bits per pixel
  • Colors = 2 n
📝 Exam Example 100×100 pixels, 8-bit depth → 100 × 100 × 8 = 80,000 bits = 80 Kbit = 10 KB
(Note: 8 bits → 1 Byte)
02
CRT Display & Interlacing
📺
CRT Components
الشرح بالعربي شاشة CRT تعمل بمسدس إلكتروني يطلق شعاعاً نحو شاشة مطلية بالفوسفور تضيء عند الاصطدام. نهاية كل سطر = Horizontal Flyback . نهاية الإطار كاملاً = Vertical Flyback .
  • Evacuated tube — vacuum chamber
  • Electron gun fires a beam of electrons
  • Focus mechanism — focuses the beam
  • Horizontal & vertical deflection plates steer the beam
  • Phosphor-coated surface glows when hit (phosphorescence)
  • End of line = horizontal flyback | End of image = vertical (frame) flyback
⚡
Interlacing
الشرح بالعربي تقنية تقليل عرض النطاق الترددي للنصف. الإطار يُعرض على مرحلتين (حقلين): الحقل الأول = الأسطر الفردية (1,3,5...) ثم الحقل الثاني = الأسطر الزوجية (0,2,4...). معدل تحديث 50 هرتز interlaced → معدل الإطارات الفعلي = 25 هرتز.
  • Halves bandwidth from memory to video controller
  • Frame displayed in 2 passes (fields)
  • 1st field: odd scan lines (1, 3, 5…)
  • 2nd field: even scan lines (0, 2, 4…)
50 Hz interlaced → 1/50 + 1/50 = 2/50 = 1/25 = 25 Hz frame rate
🌈
Color CRT — Shadow Mask
الشرح بالعربي شاشة الألوان: 3 مسدسات إلكترون (R, G, B). القناع الظلي يوجّه كل شعاع نحو نقطة الفوسفور الصحيحة. كل ثلاثي (Triad) = بكسل واحد.
  • 3 electron guns: Red, Green, Blue
  • Shadow mask directs each beam to the correct phosphor dot
  • Each phosphor triad (R+G+B dot) = one pixel

04
Image Types: Binary, Grayscale & Color
⬛
Binary Image (1-Bit)
الشرح بالعربي الصورة الثنائية: كل بكسل إما أسود (0) أو أبيض (1). تستخدم بت واحد فقط لكل بكسل.
  • 1-bit per pixel — only 2 possible values
  • 0 = black (background), 1 = white (foreground)
  • Used for text, logos, documents
🔲
Grayscale Image
الشرح بالعربي الصورة الرمادية: كل بكسل له شدة ضوء واحدة. 8 بت = 256 درجة رمادي. 2 بت = 4 درجات: 00=أسود، 01=رمادي داكن، 10=رمادي فاتح، 11=أبيض. مختلف عن الثنائية التي تحتوي على درجتين فقط.
  • Single sample per pixel = intensity
  • Range: 0 (black) → 1 (white), stored as 8-bit: 256 gray levels
  • Distinct from binary — many shades between black and white
2-Bit Grayscale Example 00 =black  |  01 =dark gray  |  10 =light gray  |  11 =white
🌈
RGB Color Image
الشرح بالعربي كل بكسل في صورة RGB يتكون من 3 قيم: Red + Green + Blue. 24 بت = 8 بت لكل قناة → 16.7 مليون لون. الصورة الملونة = ثلاث صور رمادية مدمجة.
  • Each pixel: 3 channels (R, G, B)
  • 24-bit: 8 bits × 3 = 16.7 million colors
  • Zooming in reveals individual colored squares per pixel
05
Bitmap vs Vector Formats
🗺️
Two Graphic File Formats
الشرح بالعربي Bitmap: يخزن البكسلات مباشرة — عيب: يفقد الجودة عند التكبير. Vector: يخزن الخطوط كمعادلات رياضية (نقطة بداية + طول + اتجاه) — ميزة: لا يفقد الجودة عند التكبير وحجمه أصغر (مثل MIDI للصوت).
🗺️ Bitmap (Raster)
  • Stores pixels on screen
  • ❌ Loses quality when enlarged (pixelation)
  • Formats: BMP, JPEG, PNG, GIF
📐 Vector
  • Lines stored as equations: start, length, direction
  • ✅ Scales without quality loss, smaller file size
  • Formats: SVG, AI (similar to MIDI for audio)
Image Viewing GIF, JPEG, and PNG can be viewed directly in a web browser — they are the standard internet image formats.
06
GIF Format
🎞️
GIF Characteristics
الشرح بالعربي GIF مناسب للشعارات ذات الحواف الحادة وعدد محدود من الألوان. يدعم الرسوم المتحركة. محدود بـ 256 لون فقط لذا لا يُستخدم للصور الفوتوغرافية. المصورون يستخدمون TIFF أو RAW أو JPEG.
  • Suitable for sharp-edged line art (logos) with limited colors
  • Uses lossless compression — favors flat areas of uniform color
  • Can store small animations and low-resolution film clips
  • Can be used for low-color sprite data in games
  • ❌ Max 256 colors → NOT suitable for photography
  • Photographers use TIFF, RAW, or JPEG instead

Image Analysis & Processing
🔍
Why Image Analysis is Needed
الشرح بالعربي معلومات البكسل وحدها غير كافية! لا تحتوي على معلومات عن: شكل الجسم، اتجاهه، أو التفسير ثلاثي الأبعاد. لذا نحتاج تحليل الصور. تطبيقات: كشف الأجسام، التعرف على الأنماط، تحليل المشاهد، الرؤية الحاسوبية.
  • Pixel info alone is insufficient — no info about shape, orientation, or 3D interpretation
  • Image Analysis / Image Processing is required
Applications
  • Object Detection
  • Pattern Recognition
  • Scene Analysis
  • Computer Vision
Traffic Example Questions
  • Is there a traffic sign visible?
  • Which traffic sign?
  • Is a moving car ahead?
  • What is its relative speed?

Digital Camera Pipeline
📷
How a Digital Camera Works
الشرح بالعربي الضوء يدخل من العدسة → يسقط على مصفوفة CCD التي تخزن الشدة كجهد كهربائي → محول ADC يحول الشحنة لقيمة رقمية → الصورة تُخزَّن في الذاكرة للعرض أو النقل.
Lens → CCD Array → ADC → Memory → I/O Interface
  • Shutter opened briefly to expose lens to light
  • Light falls on CCD array — intensity stored as voltage
  • Charge converted to digital value by ADC (Analog-to-Digital Converter)
  • Digital image stored in memory for viewing or transfer
📡
TV Broadcast Signal
الشرح بالعربي الإشارة المركّبة للتلفزيون تُقسَّم إلى: Luminance (الإضاءة/السطوع) + Chrominance (اللون). هذا يضمن التوافق العكسي مع أجهزة التلفزيون الأبيض والأسود القديمة.
  • Broadcast: composite signal on a single channel
  • Signal divided into luminance (brightness) + chrominance (color) components
  • Backward compatible with black & white receivers

RGB Color Model & Color Space
🎨
RGB Color Model
الشرح بالعربي نموذج RGB تضافري (Additive): الأحمر + الأخضر + الأزرق تُضاف بنسب مختلفة لإنتاج مجموعة واسعة من الألوان. يعتمد على الجهاز (Device-Dependent) — نفس القيمة قد تظهر بشكل مختلف على أجهزة مختلفة بسبب اختلاف الفوسفور أو الأصباغ.
  • R, G, B light added together → broad array of colors (additive model)
  • Main purpose: sensing, representation, and display in electronic systems
  • Device-dependent — same RGB value looks different on different devices
  • Input devices: color TV cameras, image scanners, digital cameras
  • Output devices: CRT, LCD, plasma TVs, projectors, LED displays, JumboTron
🔢
RGB Hex Representation
الشرح بالعربي كل قناة (R, G, B) تُمثَّل بقيمة من 0 إلى 255 (8 بت). في النظام الست عشري: 00 إلى FF. مثال: الأحمر الخالص = FF0000، الأبيض = FFFFFF، الأسود = 000000.
RGB Hex Examples Red = #FF0000   (R=255, G=0, B=0 ) Green = #00FF00   (R=0, G=255, B=0 ) Blue = #0000FF   (R=0, G=0, B=255) White = #FFFFFF   (R=255, G=255, B=255) Black = #000000   (R=0, G=0, B=0 )

Bitmap File Structure
🗄️
BMP File Components
الشرح بالعربي كل ملف Bitmap يتكون من ثلاثة أجزاء: (1) ترويسة الملف — النوع والحجم. (2) جدول الألوان. (3) مصفوفة البيانات — البكسلات الفعلية.
  • Header — file type and size info
  • Color Table Array
  • Bitmap Data Array — actual pixel data
📋
BITMAPINFOHEADER Fields
الشرح بالعربي ترويسة المعلومات تحتوي على: الحجم، العرض، الارتفاع، عدد طبقات الألوان، عمق الألوان، نوع الضغط، حجم الصورة، ودقة الجهاز.
Member Description
dwSize Size of BITMAPINFOHEADER structure (bytes)
dwWidth Width of bitmap (pixels)
dwHeight Height of bitmap (pixels)
wPlanes Number of planes for target device — must be 1
wBitCount Bits per pixel — must be 1, 4, 8, or 24
dwCompression BI_RGB, BI_RLE8, or BI_RLE4

dwClrImportant Important color indexes (0 = all important)

Data Compression
📦
Why Compression is Needed
الشرح بالعربي البيانات غير المضغوطة تحتاج مساحة هائلة: ثانية واحدة من صوت الهاتف = 64 كيلوبت. ثانية من صوت CD = 705.6 كيلوبت. PAL TV بدون ضغط = 216 ميجابت/ثانية! لذا الضغط ضرورة قصوى.
  • Telephone audio (8 kHz, 8-bit): 64 Kbits/sec
  • CD stereo (44.1 kHz, 16-bit): 705.6 Kbits/sec
  • PAL TV : (13.5 + 6.75 + 6.75) MHz × 8 bits = 216 Mbit/sec
📝 Video Size Example 100×100 RGB video, 24 fps, 1 minute:
100 × 100 × 24 bits × 24 frames × 60 sec ÷ 8 = 43.2 MB
⚖️
Lossless vs Lossy
الشرح بالعربي Lossless: ضغط بدون فقدان — يستعيد البيانات الأصلية بالكامل. Lossy: ضغط مع فقدان — بعض البيانات غير الضرورية تُفقد، نسبة ضغط أعلى. المثال: 25.888888888 → Lossless: 25.[9]8 | Lossy: 26.
✅ Lossless
  • Decompressed = exactly matches original
  • Entropy encoding is usually lossless
  • Required for: medical X-Rays, legal docs
  • Exploits statistical redundancy
⚠️ Lossy
  • Some data is permanently lost
  • Source encoding is usually lossy
  • JPEG "rounds off" less-important info
  • Higher compression ratio, not reversible
🗂️
Compression Classification
الشرح بالعربي ثلاثة تصنيفات: Entropy Encoding (بدون فقدان)، Source Encoding (مع فقدان)، Hybrid (دمج الاثنين — الأكثر شيوعاً في الأنظمة الحديثة).
Type Approach Loss? Examples
Entropy Data as digital sequence, no semantics Lossless RLE, Huffman, Arithmetic
Source Semantics taken into account (DCT etc.) Lossy JPEG (partial), MPEG
Hybrid Combination of entropy + source Both JPEG, MPEG, H.261, DVI
Compression Standards JPEG — still images  |  H.261 — video conferencing  |  MPEG — video + audio  |  DVI — still + continuous media (PLV & RTV modes)

Run-Length Encoding (RLE)
🔁
How RLE Works
الشرح بالعربي ترميز طول التشغيل: تسلسلات البايتات المتكررة تُستبدل بـ (العدد، القيمة). إذا تكرر بايت 4 مرات أو أكثر متتالية، يُخزَّن كـ (البايت + العلامة الخاصة + العدد - 4). يُطبَّق صفاً صفاً على الصور. أسوأ حالة = صورة عشوائية (ضوضاء) → الملف المضغوط أكبر من الأصل!
  • Content-dependent coding — sequences of identical bytes replaced by (count, value)
  • A special flag byte is used (not in the stream itself)
  • If a byte occurs ≥ 4 consecutive times : count = occurrences − 4 (offset = 4)
  • Can represent 4 to 259 bytes with just 3 bytes
  • Applied row-by-row to images as pairs (color, length)
  • Component in the JPEG compression pipeline
📋
RLE Example
Original Sequence (1 Row) BBBBWWWYYYYYYGGGGYYY RLE Encoded (B,4) , (W,3) , (Y,6) , (G,4) , (Y,4)
⚠️ Worst Case For random noise images — RLE produces a larger file than the original.
Savings are entirely dependent on data content .

Huffman Encoding
🌳
Huffman Concept
الشرح بالعربي ترميز هوفمان: طريقة إحصائية مثلى. الحروف الأكثر تكراراً تأخذ رموزاً أقصر، والأقل تكراراً تأخذ رموزاً أطول. الناتج خالٍ من البادئات (Prefix-Free) → فك الضغط فريد ومضمون. يُبنى شجرة ثنائية لتحديد الرموز.
  • Optimal code using minimum bits for given character probabilities
  • Frequent chars → shorter codes | Rare chars → longer codes
  • Successive characters coded independently
  • Resulting code is Prefix-Free → guaranteed unique decoding
  • A binary tree : leaves = characters, nodes = probabilities, edges = 0/1
🧮
Full Worked Example (A, B, C, D, E)
الشرح بالعربي B=0.51 (الأعلى) → أقصر رمز = "1". C=0.09 (الأدنى) → أطول رمز = "000". متوسط طول الرمز = 1.98 بت/رمز. 1000 حرف بـ Huffman = 248 بايت مقابل 1000 بايت بـ ASCII → توفير 75%!
Given Probabilities A: p=0.16 | B: p=0.51 | C: p=0.09 | D: p=0.13 | E: p=0.11 Huffman Codes B = 1 (shortest — most frequent) A = 010 D = 011 E = 001 C = 000 (longest — least frequent)
Average Code Length = (3×0.16) + (1×0.51) + (3×0.09) + (3×0.13) + (3×0.11) = 0.48 + 0.51 + 0.27 + 0.39 + 0.33 = 1.98 bits/symbol
📊 Compression Result — 1000 Characters Huffman: 1000 × 1.98 = 1980 bits = 248 Bytes
ASCII: 1000 × 8 bits = 8000 bits = 1000 Bytes (1 KB)
→ ~75% size reduction!
Tree Building Steps 1. Sort characters by probability (lowest first)
2. Combine two lowest → new node (sum of probabilities)
3. Repeat until one root node (probability = 1.0)
4. Assign 0 / 1 to left / right edges
5. Read code from root → leaf for each character
📝 Slide 53 Note Example: AB → 1011  |  DE → 011010
