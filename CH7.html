01
Pixels & Resolution
🖼️
What is a Digital Image?
الشرح بالعربي الصورة الرقمية تتكون من عدد كبير من عناصر الصورة تُسمى بكسل (Pixels). كلما زاد عدد البكسلات، كانت الصورة ذات دقة أعلى وجودة أفضل. طريقة تخزين الصور تُسمى Bitmap وهي نفس الطريقة التي يتم بها تخزين محتويات الشاشة في ذاكرة الفيديو.
  • A digital image consists of many picture elements (pixels).
  • The number of pixels determines image quality (resolution) — more pixels = better quality.
  • Higher resolution always yields better quality.
  • A bit-map stores graphic data the same way a monitor stores data in video memory.
📐
Resolution & Pixel Depth
الشرح بالعربي دقة الصورة = العرض × الارتفاع (بالبكسلات). عمق البكسل هو عدد البتات لكل بكسل ويحدد عدد الألوان = 2 أس عدد البتات. مثال: عمق 8 بت = 256 لون.
Resolution
  • Width × Height (in pixels)
  • e.g. 1920 × 1080 = Full HD
Pixel Depth
  • 8-bit, 16-bit, 24-bit, or n-bit
  • Colors = 2n
📝 Exam Example A picture of 100×100 pixels with pixel depth of 8 bits. What is the size?
100 × 100 × 8 = 80,000 bits = 80 Kbits = 10 KB (since 8 bits → 1 Byte)
02
CRT Display & Interlacing
📺
CRT (Cathode Ray Tube) Components
الشرح بالعربي شاشة CRT (أنبوب أشعة الكاثود) تعمل بإطلاق شعاع من الإلكترونات من مسدس الإلكترون نحو شاشة مطلية بمادة الفوسفور. عند الاصطدام تضيء نقاط الفوسفور لتكوّن الصورة. يتم تحديث الصورة بالمسح الأفقي سطراً سطراً.
  • Evacuated tube — vacuum chamber
  • Electron gun — fires electrons
  • Focus mechanism — focuses the beam
  • Horizontal & Vertical deflection plates — steer the beam
  • Phosphor-coated surface — glows when hit by electrons (phosphorescence)
  • End of line = horizontal flyback | End of image = vertical (frame) flyback
⚡
Interlacing
الشرح بالعربي الـ Interlacing تقنية لتقليل عرض النطاق الترددي إلى النصف. تعرض الإطار على مرحلتين (حقلين): أولاً السطور الفردية (1,3,5...) ثم السطور الزوجية (0,2,4...). إذا كان معدل التحديث 50 هرتز interlaced فإن معدل الإطارات الحقيقي = 25 هرتز.
  • Technique to halve bandwidth from memory to video controller
  • Frame displayed in 2 passes (fields)
  • 1st field: odd scan lines only (1, 3, 5 …)
  • 2nd field: even scan lines only (0, 2, 4 …)
  • Refresh rate effectively halved
50 Hz field refresh → 1/50 + 1/50 = 2/50 = 1/25 = 25 Hz frame rate
🌈
Color CRT — Shadow Mask
الشرح بالعربي شاشة الألوان تستخدم ثلاث مسدسات إلكترون (أحمر، أخضر، أزرق). القناع الظلي (Shadow Mask) يوجّه كل شعاع نحو نقطة الفوسفور الصحيحة (Triad). دمج R+G+B ينتج جميع الألوان.
  • 3 electron guns: Red, Green, Blue
  • Shadow mask directs each beam to the correct phosphor dot
  • Each phosphor triad = one pixel (R dot + G dot + B dot)

04
Image Types: Binary, Grayscale & Color
⬛
Binary Image
الشرح بالعربي صورة ثنائية — كل بكسل إما أسود (0) أو أبيض (1). تستخدم بت واحد فقط لكل بكسل.
  • Only 2 possible values per pixel
  • 1-bit pixel depth: 0 = black, 1 = white
  • Foreground vs background color
  • Used in documents, logos, text scanning
🔲
Grayscale Image
الشرح بالعربي صورة رمادية — كل بكسل له شدة ضوء واحدة. يستخدم عادةً 8 بت = 256 درجة من الرمادي.
  • Single sample per pixel = intensity
  • Range: 0 (black) → 1 (white)
  • Common: 8 bits = 256 gray levels
  • Distinct from binary (many shades of gray)
  • 2-bit: 00=black, 01=dark gray, 10=light gray, 11=white
🌈
RGB Color Image
الشرح بالعربي كل بكسل في صورة RGB يتكون من ثلاث قيم: Red (أحمر) + Green (أخضر) + Blue (أزرق). دمج القيم الثلاثة ينتج الألوان المختلفة. اللون الحقيقي 24-bit = 8 bits لكل قناة.
  • Each pixel has 3 channels: R, G, B
  • 24-bit: 8 bits per channel → 16.7 million colors
  • Adding R+G+B light = additive color model
  • Each channel: 0–255 intensity values
05
Image Formats: Bitmap vs Vector & File Types
🗺️ Bitmap (Raster)
  • Stores pixels on screen
  • Disadvantage: loses quality when enlarged (pixelation)
  • Examples: BMP, JPEG, PNG, GIF
📐 Vector
  • Lines stored as equations: start point, length, direction
  • Advantage: scales without quality loss, smaller size
  • Examples: SVG, AI, EPS (similar to MIDI for audio)
📄
GIF Format Details
الشرح بالعربي صيغة GIF مناسبة للصور ذات الحواف الحادة كالشعارات. تدعم الرسوم المتحركة. محدودة بـ 256 لون فقط لذا لا تُستخدم للصور الفوتوغرافية.
  • Suitable for sharp-edged line art (logos) with limited colors
  • Uses lossless compression — favors flat uniform color areas
  • Can store small animations and low-resolution film clips
  • Can be used for low-color sprite data in games
  • Limitation: max 256 colors → NOT suitable for photography
  • Digital photographers use TIFF, RAW, or JPEG instead
📌 Viewing Images GIF, JPEG, and PNG can be viewed directly in a web browser — they are the standard internet image formats.

Image Analysis & Digital Camera
🔍
Image Analysis (Image Processing)
الشرح بالعربي معلومات البكسل وحدها غير كافية! نحتاج تحليل الصور لمعرفة شكل الأجسام واتجاهها والتفسير ثلاثي الأبعاد. التطبيقات تشمل: كشف الأجسام، التعرف على الأنماط، تحليل المشاهد، والرؤية الحاسوبية.
  • Pixel info alone is insufficient — no info about shape, orientation, or 3D interpretation
  • Image Analysis / Image Processing is required for deeper understanding
Applications
  • Object Detection
  • Pattern Recognition
  • Scene Analysis
  • Computer Vision
Traffic Example Problems
  • Is there a traffic sign visible?
  • Which traffic sign?
  • Is a moving car ahead?
  • What relative speed?

Digital Camera Pipeline
الشرح بالعربي الكاميرا الرقمية: الضوء يدخل من العدسة → يسقط على مصفوفة CCD حيث تُخزَّن الشدة كجهد كهربائي → يحوّل ADC الشحنة لقيمة رقمية → تُخزَّن الصورة في الذاكرة للعرض أو النقل.
Lens → CCD Array → ADC → Memory → I/O Interface
  • Shutter opened briefly to expose lens to light
  • Light falls on CCD array — intensity stored as voltage
  • Charge converted to digital value by ADC (Analog-to-Digital Converter)
  • Digital image stored in memory for viewing or transfer

Color Spaces — RGB Model
🎨
RGB Color Model
الشرح بالعربي نموذج RGB: الأحمر + الأخضر + الأزرق تُضاف معاً بنسب مختلفة لإنتاج مجموعة واسعة من الألوان. هو نموذج تضافري (Additive). الغرض الأساسي: الاستشعار والتمثيل والعرض في الأنظمة الإلكترونية. هو نموذج يعتمد على الجهاز — نفس قيمة RGB قد تظهر بألوان مختلفة على أجهزة مختلفة.
  • Red, Green, Blue light added together to reproduce a broad array of colors
  • Main purpose: sensing, representation, and display in electronic systems (TVs, computers)
  • Device-dependent — same RGB value looks different on different devices (different phosphors/dyes)
  • Input devices: color TV cameras, scanners, digital cameras
  • Output devices: CRT, LCD, plasma TVs, monitors, projectors, LED displays, JumboTron
📝 Exam Tip In TV broadcast, signal is divided into luminance (brightness) and chrominance (color) components — this allows backward compatibility with black & white receivers.

🗄️
BMP File Components
الشرح بالعربي كل ملف Bitmap يحتوي على ثلاثة أجزاء: (1) ترويسة الملف (File Header) — نوع وحجم الملف. (2) جدول الألوان (Color Table Array). (3) مصفوفة البيانات — بيانات البكسلات الفعلية.

Data Compression — Overview
📦
Why Compression is Needed
الشرح بالعربي البيانات غير المضغوطة تحتاج مساحة هائلة. ثانية واحدة من الصوت الهاتفي = 64 كيلوبت. ثانية من صوت CD = 705.6 كيلوبت. الفيديو PAL يحتاج 216 ميجابت/ثانية! لذا الضغط ضرورة لا رفاهية.
  • Telephone audio (8 kHz, 8-bit): 64 Kbits per second
  • CD stereo (44.1 kHz, 16-bit): 705.6 Kbits per second
  • PAL TV: 625 lines, 25 fps → 216 Mbit/sec uncompressed!
📝 Video Size Example 100×100 RGB video at 24 fps for 1 minute:
100 × 100 × 24 bits × 24 frames × 60 seconds ÷ 8 = 43.2 MB
✅
Lossless Compression
الشرح بالعربي ضغط بدون فقدان — يمكن استعادة البيانات الأصلية بالكامل.
  • Decompressed data exactly matches original
  • Entropy encoding is usually lossless
  • Required for: medical images (X-Rays), legal docs
  • Exploits statistical redundancy
  • Example: English 'e' is more common than 'z'
⚠️
Lossy Compression
الشرح بالعربي ضغط مع فقدان — بعض البيانات غير الضرورية تُفقد ولا يمكن استعادتها. يعطي نسب ضغط أعلى.
  • Some insignificant data is lost when decompressing
  • Source encoding is usually lossy
  • JPEG "rounds off" less-important information
  • Higher compression ratio than lossless
  • Not reversible
📝 Classic Example String: 25.888888888 → Lossless: 25.[9]8 (exact recovery) | Lossy: store as 26 (data lost but smaller)
🗂️
Compression Classification
الشرح بالعربي ثلاثة أنواع: ترميز الإنتروبيا (Entropy Encoding) — يعامل البيانات كتسلسل رقمي بدون معنى — بدون فقدان. ترميز المصدر (Source Encoding) — يأخذ معنى البيانات بعين الاعتبار — مع فقدان. ترميز هجين (Hybrid) — الأكثر شيوعاً.
Type Approach Loss? Examples
Entropy Encoding Data treated as digital sequence, no semantics Lossless RLE, Huffman, Arithmetic
Source Encoding Semantics of data taken into account (DCT etc.) Lossy JPEG (partial), MPEG
Hybrid Encoding Combination of entropy + source encoding Both JPEG, MPEG, H.261, DVI
Compression Standards
  • JPEG — still images
  • H.261 — video conferencing (ISDN/mobile)
  • MPEG — video and audio
  • DVI — still + continuous media (PLV & RTV modes)
PAL TV Bandwidth
  • 625 lines, 25 fps
  • Luminance: 13.5 MHz
  • Chrominance R-Y: 6.75 MHz
  • Chrominance B-Y: 6.75 MHz
  • Total: 216 Mbit/sec

Run-Length Encoding (RLE)
🔁
How RLE Works
الشرح بالعربي ترميز طول التشغيل: تسلسلات البايتات المتكررة تُستبدل بعدد تكراراتها. بدلاً من تخزين AAAA نخزن (A,4). يُستخدم كثيراً في ضغط الصور، وهو جزء من pipeline ضغط JPEG.
  • Content-dependent coding technique
  • Sequences of identical bytes/bits replaced by (count, value)
  • A special flag byte is used (one not in the stream)
  • If a byte occurs ≥ 4 consecutive times, count = occurrences − 4 (offset = 4)
  • Can represent 4 to 259 bytes with just 3 bytes
  • Applied row-by-row to images: mapped to pairs (color, length)
📋
RLE Example
الشرح بالعربي مثال: الصف BBBBWWWYYYYYYGGGGYYY يُرمَّز كـ: (B,4),(W,3),(Y,6),(G,4),(Y,4). في أسوأ حالة (ضوضاء عشوائية) يكون الضغط أثقل من الأصل!
Original Sequence (1 Row) BBBBWWWYYYYYYGGGGYYY

RLE Encoded (B,4) , (W,3) , (Y,6) , (G,4) , (Y,4)
⚠️ Worst Case For random noise images — RLE encoding can produce a larger file than the original! The savings depend entirely on the data content.

Huffman Encoding
🌳
Huffman Concept
الشرح بالعربي ترميز هوفمان: طريقة إحصائية للضغط. الحروف الأكثر تكراراً تأخذ رموزاً أقصر، والأقل تكراراً تأخذ رموزاً أطول. الناتج خالٍ من البادئات (Prefix-Free) مما يضمن فك الضغط الصحيح. يُبنى شجرة ثنائية لتحديد الرموز.
  • Optimal code using minimum bits for given character probabilities
  • Statistical encoding: sender and receiver both know occurrence probabilities
  • Frequent chars → shorter codes | Rare chars → longer codes
  • Successive characters coded independently
  • Resulting code is Prefix-Free → unique decoding guaranteed
  • A binary tree is built: leaves = characters, nodes = probabilities, edges = 0/1
🧮
Huffman Example — Full Worked Solution
الشرح بالعربي للحروف A,B,C,D,E مع احتمالياتها: B=0.51 (الأعلى) يأخذ أقصر رمز = "1". C=0.09 (الأدنى) يأخذ أطول رمز = "000". متوسط طول الرمز = 1.98 بت/رمز، مقارنةً بـ 8 بت في ASCII.
Given Probabilities A: p=0.16  |  B: p=0.51  |  C: p=0.09  |  D: p=0.13  |  E: p=0.11
Huffman Codes B = 1      (shortest — most frequent)
A = 010
D = 011
E = 001
C = 000   (longest — least frequent)
Average Code Length Calculation = (3×0.16) + (1×0.51) + (3×0.09) + (3×0.13) + (3×0.11)
= 0.48 + 0.51 + 0.27 + 0.39 + 0.33
= 1.98 bits/symbol
📊 Compression Comparison — 1000 Characters Huffman: 1000 × 1.98 = 1980 bits ≈ 248 Bytes
ASCII: 1000 × 8 bits = 8000 bits = 1000 Bytes (1 KB)
Saving: ~75% size reduction!
📝 Tree Building Steps 1. Sort characters by probability (lowest first)
2. Combine two lowest probabilities into a new node
3. Repeat until one root node (probability = 1.0)
4. Assign 0 and 1 to left/right edges
5. Read code from root → leaf for each character
